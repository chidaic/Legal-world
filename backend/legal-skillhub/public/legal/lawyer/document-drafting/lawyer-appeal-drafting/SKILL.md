# 目标

根据玩家律师提供的提示、草稿、一审裁判信息和案件上下文，辅助生成可供玩家继续编辑确认的《民事上诉状》正文。

本 Skill 只规定文书内容、结构和质量要求。玩家最终确认、保存结果文件和导出 PDF 由后端流程处理，不要在正文中写工具调用、PDF 路径、导出说明或完成提示。

# 生成规则

1. 只输出完整《民事上诉状》正文，不输出解释、摘要、Markdown 标题、代码块或操作说明。
2. 以玩家提示和玩家草稿为主要意图来源；案件上下文用于补足一审裁判、主体、上诉请求、上诉理由、新证据和法院信息。
3. 不要编造一审判决内容、事实、证据、金额、日期、地址、身份证号或主体信息。
4. 信息缺失时，在对应位置写“待补充”，不要为了完整而虚构。
5. 上诉请求必须明确指向一审裁判结果，例如撤销、改判、发回重审或调整具体给付责任。
6. 上诉理由必须与上诉请求对应，围绕事实认定、证据采信、法律适用、程序问题或裁判结果不当展开。
7. 如果玩家草稿已有明确表达，优先保留其上诉请求和理由主线，只做结构化、法律化和格式化整理。

# 二审证据边界

- 《民事上诉状》中只写二审阶段新增证据。
- 一审已经提交、质证或使用过的证据，不要作为二审新证据清单写入上诉状。
- 如需说明一审漏审、误采或未充分采信证据，应放在“事实和理由”中作为一审裁判错误论证，不要整理成新的证据目录。
- 如果玩家明确没有二审新证据，或上下文只显示继续使用一审证据，正文的“二审新证据”部分写“无二审新证据。”。
- 新证据名称、来源或证明目的不明时写“待补充”。

# 文书模板

正文必须按下列模板组织：

```text
                    民事上诉状

上诉人（自然人）：XXX，男/女，XXXX年XX月XX日生，X族，住XXXXXX。
上诉人（法人）：XXX，法定代表人：XXX，住所：XXXXXX。

被上诉人（自然人）：XXX，男/女，XXXX年XX月XX日生，X族，住XXXXXX。
被上诉人（法人）：XXX，法定代表人：XXX，住所：XXXXXX。

上诉请求：
1. 上诉请求1。
2. 上诉请求2。
...

事实和理由：
...

二审新证据：
1. 新证据名称、来源及证明目的。
如无二审新证据，写：无二审新证据。

此致
[受诉法院名称]
```

# 质量检查

输出前检查：

- 标题是否为“民事上诉状”。
- 上诉人、被上诉人、上诉请求、事实和理由、二审新证据、受诉法院是否齐全。
- 上诉请求与上诉理由是否对应。
- 是否把一审旧证据错误列为二审新证据。
- 是否误写了工具调用、PDF、文件路径、后台导出或“起草结束”等流程性内容。
- 是否存在没有来源的事实、金额、日期、证据或主体信息。

# 兼容说明

如果本 Skill 被自动 Agent 场景加载，是否调用文书工具由场景系统规则决定；本 Skill 不重复规定工具调用流程。玩家律师辅助接口使用本 Skill 时，只把它作为正文生成规范。
